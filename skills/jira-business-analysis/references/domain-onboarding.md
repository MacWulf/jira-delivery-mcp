# Domain Onboarding

Use this flow when the domain is unfamiliar, specialized, or regulated.

## Source order

1. repo docs
2. Jira context
3. Confluence context
4. targeted web research

## Research goal

Research only enough to:
- understand the core terminology
- identify common actors and process steps
- recognize likely constraints or risk areas
- ask better questions

Do not use research to replace the user's actual operating model.

## Trigger signals for web research

- heavy domain terminology
- regulated or compliance-heavy area
- ambiguous industry jargon
- user request implies domain nuance that generic software patterns would miss

## Output of onboarding

Capture:
- domain summary
- key terms
- likely actors
- likely risk areas
- questions that research resolved
- questions still worth asking the user
