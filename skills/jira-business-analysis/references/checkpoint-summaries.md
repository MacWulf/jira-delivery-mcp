# Checkpoint Summaries

Create a checkpoint after every 5 to 10 questions, or when a topic block closes early.

## Required shape

- `What we learned`
- `Assumptions currently in use`
- `Open questions`
- `Conflicts or ambiguities`
- `Do not ask again unless contradicted`

## Purpose

- prevent repetition
- preserve incremental knowledge
- tighten the next question plan
- make stopping conditions visible

## Usage rule

Before asking the next question, review the latest checkpoint and avoid reopening closed topics unless the user contradicted them.
Stakeholder presence is a closed topic after the first clear answer and must not be asked again unless later input directly conflicts with it.
